# Repository scope

## Intended role

Public-safe case studies, project narratives, artifacts, and documented outcomes from CrossingKey.

## Human-readable standard

Documentation and source must use explicit names, modest dependencies, versioned decisions, and clear setup instructions. Material should remain understandable and maintainable without relying on unstated context.

## Exclusions

No secrets, access tokens, private customer information, or sensitive production configuration may be committed to this repository.
